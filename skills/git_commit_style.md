# When writing a git commit message

- Use conventional commits format: `type(scope): summary`
- Types: feat, fix, refactor, chore, docs, test
- Summary line under 72 chars, imperative mood ("add", not "added")
- If the change is non-trivial, add a body explaining *why*, not just *what*
- Never mention "AI" or "Claude" or similar in the commit message
